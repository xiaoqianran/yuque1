## 2. 设计原则与约束

| 原则 | 落地 |
|------|------|
| 对齐 PRD 冻结决策 | 手机号强制验证；乐观锁冲突三选一；folder/doc 同树；**doc 必须可挂子节点** |
| 模块化单体优先 | MVP **单进程 API**，按领域分包；禁止过早微服务；**无独立 API Gateway 进程** |
| 契约优先 | **`docs/api/openapi.yaml` 为 SSOT**；实现必须通过契约校验（见 §4.1） |
| 安全默认 | 私有默认、HttpOnly 会话、短信/全局限流、MD 防 XSS、统一 `access` 鉴权 |
| 可自托管 | Docker Compose 一键 demo；配置与密钥外置 |
| 可演进 | 预留 revision、审计、搜索模块扩展点；**不上 MQ 也能做 MVP** |
| 标识统一 | 对外资源 ID：**`tree_nodes.id`（nodeId）**；type=doc 才有正文 |

### 2.1 规模假设（设计容量，非 SLA 承诺）

| 项 | 假设 |
|----|------|
| 用户 | ≤ 500 / 单部署 |
| 文档 | ≤ 5 万篇 |
| 单篇 Markdown | 常规 &lt; 100KB；**硬上限 2MB**（超限 HTTP 413） |
| 单库树节点 | ≤ 5,000；**MVP 全量拉树建议 ≤ 1,000**；超限预留懒加载 |
| 并发写同一文档 | 低；靠乐观锁，非实时协同 |
| 租户模型 | MVP **单部署单逻辑租户**；不建 `tenant_id` |

### 2.2 缓存原则

| 数据 | MVP 策略 |
|------|----------|
| Session / 短信码 / 限流计数 | Redis |
| 知识库树、正文、权限 | **默认不缓存**（避免私有文档脏读与越权缓存） |
| 一期可选 | 公开分享页短 TTL 缓存（须与 `enabled`/吊销强一致） |

## 3. 系统架构

### 3.1 逻辑架构

```
┌─────────────────────────────────────────────────────────────┐
│                        Client (Web)                         │
│  React + TS + Vite │ Editor(MD) │ Tree │ Auth │ Share View  │
└───────────────────────────┬─────────────────────────────────┘
                            │ HTTPS / JSON (REST)
                            │ Cookie: sid (HttpOnly)  同站同源
┌───────────────────────────▼─────────────────────────────────┐
│              API 模块化单体 (NestJS, 单进程)                   │
│  HTTP 中间件管线（非独立网关）:                                 │
│    RequestId → RateLimit → SessionAuth → ...                 │
│  Guards/Interceptors: access 鉴权、审计埋点、统一异常          │
│  ┌────────┬──────────┬─────┬──────┬──────────┬───────────┐   │
│  │identity│workspace*│ kb  │ tree │ document │   share   │   │
│  │  sms   │ (内部)   │     │      │ content  │   media†  │   │
│  └────────┴──────────┴─────┴──┬───┴──────────┴───────────┘   │
│                               │                                │
│                    ┌──────────▼──────────┐                     │
│                    │  access（授权中心）   │  所有资源读写必经    │
│                    └─────────────────────┘                     │
│  audit / health / （预留 search）                               │
└───────┬─────────────┬──────────────┬──────────────┬───────────┘
        ▼             ▼              ▼              ▼
   PostgreSQL      Redis         SMS Provider    MinIO†
                 session/                        (图片 P1)
                 sms/ratelimit
```

\* `workspace`：MVP **仅内部归属**，不暴露多空间 CRUD API（见 §5）。  
† Media / MinIO：P1；MVP 图片策略见 §6.7。

### 3.2 部署架构

#### 3.2.1 浏览器访问形态（冻结）

| 项 | MVP / 生产冻结决策 |
|----|-------------------|
| 访问形态 | **同站同源**：`https://{domain}/` 静态资源；`https://{domain}/api/v1/*` 反代到 API |
| 禁止 | 生产环境前后端双域名且无统一同站入口（避免 Cookie/CSRF 分叉） |
| local 例外 | Vite `localhost:5173` → API `localhost:3000`，CORS 白名单 + `credentials` |
| Cookie | 名 `sid`；`HttpOnly; Secure(prod); SameSite=Lax; Path=/` |
| CSRF | **同站部署下 MVP 不强制 Double-Submit Token**；若未来 API 独立站点，必须 CSRF 或改 Bearer |
| CORS | 生产默认不开放跨域写；仅 local 配置 `CORS_ORIGIN` |
| API 扩展 | Session **仅存 Redis**，API 无本地会话，可多副本水平扩展 |

#### 3.2.2 拓扑

**local / demo（Compose）**

```
┌──────────────────────────────────────────┐
│              Docker Compose              │
│  web(nginx 静态+/api 反代) → api(node22) │
│       → postgres │ redis │ minio(opt)    │
└──────────────────────────────────────────┘
```

| 环境 | 说明 |
|------|------|
| local | Compose + 短信 Mock；CORS 指向 Vite |
| dev/staging | 独立 DB/Redis；短信 Mock 或真网关 |
| prod 最低建议 | **API ≥ 2 副本**；PostgreSQL 托管或主从；Redis 持久化；TLS 终止；资源 limit；**勿把单机 Compose 当生产高可用** |

### 3.3 请求链路

1. 浏览器 → Nginx（静态 + `/api` 反代）  
2. 中间件：`requestId` → 限流 → Session 解析  
3. Guard：`access.assert(...)`（分享通道见 §6.4）  
4. Application Service 事务编排  
5. 按规则写 `audit_logs` / `content_revisions`  
6. 统一错误体 + 语义化 HTTP 状态码（**禁止业务错误一律 200**）  

### 3.4 异步与消息（MVP）

- MVP **全部同步**处理（保存、树变更、分享）。  
- **不上 MQ**。  
- 扩展点：一期搜索/通知可增加 `outbox` 表或模块内异步，不必先上 Kafka。

## 5. 后端模块划分

路径示例：`apps/api/src/modules/*`

| 模块 | 职责 | MVP |
|------|------|:---:|
| `identity` | 用户（E.164 手机号）、短信、会话、可选密码(P1)、资料 | ✅ |
| `workspace` | 注册时创建个人空间；**无对外 CRUD** | ✅ 内部 |
| `knowledge-base` | 知识库 CRUD；成员(P1) | ✅ / 成员 P1 |
| `tree` | TreeNode：创建/移动/排序/删除；**parent 可为 doc** | ✅ |
| `document` | 正文读写、版本、冲突三路径、revision | ✅ |
| `share` | 分享 token 生成/校验/吊销 | ✅ |
| **`access`** | **统一授权**：kb/node 读写下 admin；分享只读通道 | ✅ |
| `audit` | 审计写入（P0 事件必写） | ✅ 最小 |
| `search` | 标题过滤（可先落在 tree/kb 内，逻辑边界独立） | ✅ 最小 |
| `media` | 上传与元数据 | P1 |
| `health` | `/health`、`/ready` | ✅ |

### 5.1 依赖方向（禁止循环）

```
access ← 被 document/tree/kb/share 依赖（只提供鉴权，不依赖业务写）
document → tree → knowledge-base → workspace → identity
share → document + access
audit ← 被多模块调用（单向）
search → tree + access
```

说明：`access` **不得**依赖 `document` 的写服务；仅依赖仓储只读查询（角色、owner、share）。

### 5.2 Workspace 决策（冻结）

- 表可保留 `workspaces`（user 1:1 personal）。  
- **API 不暴露** `/workspaces` CRUD。  
- 知识库直接归属 `workspace_id`（内部），用户侧只感知「我的知识库」。

## 6. 核心领域机制

### 6.1 身份与短信登录

**手机号**：存储与校验统一 **E.164**（例：`+86138****`）。

```
客户端 输入手机号 → POST /api/v1/auth/sms/send
API:
  1. 格式校验 + 限流
  2. 生成 6 位码
  3. SmsProvider.send
  4. 仅当发送成功（或 Mock 成功）→ Redis SET sms:{e164}（建议存 hash(code)）TTL 5min
  5. 发送失败不得留下有效码

客户端 提交手机号+码(+昵称) → POST /api/v1/auth/sms/login
API:
  校验码（成功则 DEL）→ 无用户则创建 User(mobile_verified=true)+Workspace
  → 创建 Session（建议登录时轮换 sid，防固定会话）→ Set-Cookie
```

| 规则 | 值（可配置，下列为默认） |
|------|--------------------------|
| 验证码长度 | 6 位数字 |
| TTL | 5 分钟 |
| 发送间隔 | ≥ 60 秒 / 手机号 |
| 日上限 | 20 次/手机；50 次/IP |
| 校验失败 | 5 次 / 15 分钟锁定 |
| Session 空闲/绝对 TTL | 7 天（可配置 7～14） |
| 并发会话 | MVP **允许多端**；不强制踢旧（二期可做） |

**P1 密码**：`password_hash` 可选；验证码登录始终可用；重置走手机验证码。  
**邮箱**：可空，无验证流（P1 资料）。

### 6.2 文档树（folder / doc，doc 可挂子）

**ID 冻结**：对外只使用 **`nodeId` = `tree_nodes.id`**。  
`type=doc` 时在 `document_contents` 1:1 存在正文；**不另引入对外 documents.id**。

```
tree_nodes
  id, knowledge_base_id, parent_id NULLABLE,
  type: 'folder' | 'doc',
  title, sort_order, deleted_at, created_at, updated_at, ...
```

**硬性约束**：

1. `parent_id` 可为 `NULL`（库根）或同库任意**未删除**节点（folder **或** doc）。  
2. 创建/移动：同库、parent 未删、**不成环**（沿 parent 链上溯不得碰到自身）。  
3. `folder` 禁止有 `document_contents`。  
4. 删除：存在未删除子节点 → `409 NODE_HAS_CHILDREN`。  
5. 树查询：**始终 `deleted_at IS NULL`**；分享目标已删 → 无效。  
6. 全量树：`GET .../tree`；节点量大时一期懒加载（接口预留）。

**API（冻结前缀 `/api/v1`）**：

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/kbs/:kbId/tree` | 整树（未删除） |
| POST | `/kbs/:kbId/nodes` | `{ type, title, parentId? }` |
| PATCH | `/nodes/:nodeId` | 重命名等 |
| POST | `/nodes/:nodeId/move` | `{ parentId, sortOrder }` |
| DELETE | `/nodes/:nodeId` | 软删，非空拒绝 |

### 6.3 正文保存与冲突（企业级乐观锁）— v1.2 重写

#### 6.3.1 表要点

```
document_contents
  node_id PK/FK → tree_nodes(id)  -- 即对外 nodeId
  body_md TEXT
  content_version INT NOT NULL     -- 从 1 起
  updated_by, updated_at

content_revisions
  id, node_id, version, body_md, created_by, created_at, reason
  -- MVP 仅 reason = 'overwrite_on_conflict'
```

#### 6.3.2 默认实现：条件更新（推荐，Prisma 友好）

**正常保存** `PUT /api/v1/nodes/:nodeId/content`

```text
Body: { expectedVersion, bodyMd }

UPDATE document_contents
SET body_md = $body, content_version = content_version + 1,
    updated_by = $user, updated_at = now()
WHERE node_id = $id AND content_version = $expectedVersion
  AND EXISTS (node type=doc and not deleted)

IF affected_rows = 0:
  -- 再读当前版本，区分 NOT_FOUND 与 CONFLICT
  RETURN 409 DOC_VERSION_CONFLICT { serverVersion, updatedBy, updatedAt }
ELSE
  RETURN { version: expectedVersion+1, updatedAt }
```

事务内可用 `SELECT` 校验 `access.write`；**不依赖** Prisma 原生 `FOR UPDATE` 作为默认。

可选增强：高竞争下使用 `$queryRaw` + `FOR UPDATE`，非 MVP 必选。

#### 6.3.3 强制覆盖（冲突面板 B）

`POST /api/v1/nodes/:nodeId/content/overwrite`

```text
Body: { bodyMd }  -- 不要求 expectedVersion 匹配

BEGIN
  assert access.write
  row = 当前正文（不存在则 404）
  INSERT content_revisions(node_id, version=row.version, body_md=row.body_md,
         reason='overwrite_on_conflict', created_by=userId)
  INSERT audit_log(type='doc.overwrite_on_conflict', ...)
  UPDATE body_md, content_version = row.version+1, updated_by, updated_at
COMMIT
返回新 version
```

#### 6.3.4 另存副本（冲突面板 C）

`POST /api/v1/nodes/:nodeId/content/save-as`

```text
Body: { bodyMd, parentId?, title? }
- parentId 默认 = 源节点 parentId（与源同级）；允许指定
- title 默认 = 「{原标题} (冲突副本)」
- 新建 type=doc 节点 + document_contents version=1
- 不修改源文档
```

#### 6.3.5 拉取最新

`GET /api/v1/nodes/:nodeId/content` → `{ bodyMd, version, updatedAt, updatedBy }`

#### 6.3.6 接口一览（与旧版 `/docs` 路径废弃）

| 接口 | 用途 |
|------|------|
| `PUT /nodes/:nodeId/content` | 正常保存（期望版本匹配） |
| `POST /nodes/:nodeId/content/overwrite` | 冲突后强制覆盖 + revision + audit |
| `POST /nodes/:nodeId/content/save-as` | 另存副本 |
| `GET /nodes/:nodeId/content` | 加载最新 |

#### 6.3.7 前端状态

- 防抖 2～5s；**in-flight 合并为最后一版**（防乱序 409）。  
- 状态：`saved | saving | dirty | conflict`。  
- `conflict`：**停止自动保存** → 三选一面板。  
- 正文 &gt; 2MB：前端拦截 + 服务端 413。

### 6.4 分享链接

```
share_links
  id, token (CSPRNG ≥ 128 bit), node_id (doc only),
  enabled, expires_at NULL, created_by, created_at
```

- API：`GET /api/v1/share/:token` → 只读标题 + **Markdown 正文**（不返回未消毒 HTML）  
- 浏览器页：`/s/:token`（前端路由）  
- 校验：enabled、未过期(P1)、目标 node 未删除、type=doc  
- 关闭：`enabled=false`  
- **不**附带整棵子树读权限  
- 限流：见 §10.2  

### 6.5 授权中心 `access`（冻结）

所有资源读/写 **禁止** Controller 内散落 `if (ownerId===userId)`，必须：

```text
access.assert(actor, resource, action)
// actor = { type:'user', userId } | { type:'share', token }
// action = 'read' | 'write' | 'admin'
```

#### 6.5.1 角色（知识库级）

MVP 简化：`owner` | `editor` | `reader`（Admin 与 Owner 合并）。  
无成员时：仅创建者视为 `owner`。

| 动作 | Guest | share token | reader | editor | owner |
|------|:-----:|:-----------:|:------:|:------:|:-----:|
| 读库/树/正文 | ❌ | 仅 token 对应 doc | ✅ | ✅ | ✅ |
| 写正文/树 | ❌ | ❌ | ❌ | ✅ | ✅ |
| 分享开/关 | ❌ | ❌ | ❌ | ✅ | ✅ |
| 成员与删库 | ❌ | ❌ | ❌ | ❌ | ✅ |

响应策略：无权限与不存在 **默认统一 404**（`NOT_FOUND`，可配置），防枚举。

### 6.6 MVP 检索（标题过滤）

```
GET /api/v1/kbs/:kbId/nodes?q=keyword&limit=50
```

- 条件：`deleted_at IS NULL` AND `title ILIKE %keyword%`（注意转义）  
- 需 `access.read` on kb  
- 索引建议：`(knowledge_base_id)` + 标题检索；可选 `pg_trgm` 一期  
- **不上 ES**  

### 6.7 渲染与图片策略（冻结）

| 项 | 决策 |
|----|------|
| 渲染库 | **仅 markdown-it** |
| API 正文 | **只返回 Markdown**，不返回 HTML |
| HTML 生成 | 前端统一 `renderMarkdown` → **DOMPurify** |
| 原始 HTML | 默认关闭或极严白名单 |
| MVP 图片 | **允许 https 外链**；**不托管上传**（P1 MinIO） |
| 禁止 | 大体积 data-URL 写入 `body_md`（可校验体积） |

编辑器 POC 仍验证图片 **语法** 与外链展示；托管上传不阻塞 MVP。

### 6.8 审计范围矩阵

| 事件 | 优先级 | 说明 |
|------|:------:|------|
| `doc.overwrite_on_conflict` | **P0** | 强制覆盖 |
| `share.enable` / `share.disable` | **P0** | 分享开关 |
| `node.delete` / `kb.delete` | **P0** | 删除 |
| `auth.login_failed` / 锁定 | **P0** | 安全 |
| 全量写操作流水与审计 UI | P1 | — |

字段建议：`id, actor_id, action, resource_type, resource_id, metadata jsonb, request_id, created_at`。  
日志脱敏：手机号中间四位；保留期建议 ≥ 30 天（可配置）。

## 7. API 约定

### 7.1 风格

- REST + JSON  
- **统一前缀**：`/api/v1`  
- 鉴权：Cookie Session；`GET /share/:token` 例外  
- 分页：`?page=&pageSize=`  
- 时间：ISO-8601 UTC  
- **HTTP 状态码与 `error.code` 语义一致**  

### 7.2 统一响应

成功：

```json
{
  "success": true,
  "data": {},
  "error": null,
  "requestId": "01J..."
}
```

失败：

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "DOC_VERSION_CONFLICT",
    "message": "文档已被他人更新",
    "details": {
      "serverVersion": 4,
      "updatedAt": "2026-07-12T10:00:00.000Z",
      "updatedBy": { "id": "...", "nickname": "..." }
    }
  },
  "requestId": "01J..."
}
```

### 7.3 关键错误码

| code | HTTP | 含义 |
|------|------|------|
| `UNAUTHORIZED` | 401 | 未登录 |
| `NOT_FOUND` | 404 | 不存在或无权限（模糊策略） |
| `VALIDATION_ERROR` | 400 | 参数错误 |
| `PAYLOAD_TOO_LARGE` | 413 | 正文超 2MB |
| `SMS_RATE_LIMITED` | 429 | 短信过频 |
| `SMS_CODE_INVALID` | 400 | 验证码错误 |
| `AUTH_LOCKED` | 429 | 登录锁定 |
| `DOC_VERSION_CONFLICT` | 409 | 正文版本冲突 |
| `NODE_HAS_CHILDREN` | 409 | 非空不可删 |
| `NODE_CYCLE_DETECTED` | 400 | 移动成环 |
| `SHARE_INVALID` | 404 | 分享无效 |
| `RATE_LIMITED` | 429 | 通用限流 |

### 8.1 目录（示意）

### 8.2 编辑器 POC 标准

**MVP：CodeMirror 6 源码编辑 + 分栏/切换预览（markdown-it + DOMPurify）。**

| POC 验收项 | 通过标准 |
|------------|----------|

## 10. 安全设计

### 10.1 基线

| 域 | 措施 |
|----|------|
| 传输 | 生产全站 HTTPS |
| 会话 | HttpOnly Cookie；同站策略见 §3.2 |
| 短信 | 频控、单次失效、E.164 |
| 注入 | ORM 参数化 |
| XSS | API 只出 MD；前端消毒；CSP 头 |
| 分享 | CSPRNG token ≥128 bit；限流；可吊销 |
| 上传 P1 | MIME/大小白名单；随机对象键 |
| 密钥 | 环境变量；不进 Git；可轮转 `SESSION_SECRET` |
| 依赖 | CI 依赖漏洞扫描（基线） |
| 审计 | §6.8 P0 事件 |

### 10.2 限流默认值

| 维度 | 默认 |
|------|------|
| 短信发送 | 1/手机/60s；20/手机/日；50/IP/日 |
| 登录失败 | 5 次/15min → 锁定 |
| 分享 GET | 60 次/分钟/IP |
| 登录用户写 API | 120 次/分钟/用户 |

实现：Redis 固定窗口或滑动窗口；key 前缀 `rl:{scope}:{id}`。

### 10.3 越权测试清单（提测必跑）

- 他人 `kbId`/`nodeId` 遍历  
- 未分享 doc 直链  
- 失效/关闭 token  
- reader 写操作（P1）  
- 软删后分享与树  
- 跨库 `parentId` 移动

## 13. 关键时序

### 13.1 短信登录

```
User → Web → API → Redis / SMS
send:  限流 → SMS 成功 → SET code
login: GET code 校验 → DEL → user/session → Set-Cookie
```

### 13.2 保存冲突

```
TabB 保存成功 version=4
TabA PUT expected=3 → 409
面板:
  加载最新 → GET content
  覆盖 → POST .../overwrite → revision + version=5
  另存 → POST .../save-as → 新 nodeId
```

### 13.3 doc 下建子 doc

```
POST /kbs/{kbId}/nodes
{ "type":"doc", "title":"子篇", "parentId":"{父nodeId}" }
→ access.write → 插 tree_nodes → 插 document_contents version=1
```
