## 2. 领域总览

### 2.1 限界上下文与聚合根（v1.1）

| 上下文 | 聚合根 (AR) | 从属实体 / VO | 说明 |
|--------|-------------|---------------|------|
| Identity | **User** | VO: MobileE164, Nickname | Session 在 Redis |
| Workspace | **Workspace** | — | 1:1 User；无对外 CRUD；可视为 Identity 配套 |
| Knowledge | **KnowledgeBase** | Entity: KbMember；VO: MemberRole | **权限真源 = members** |
| TreeDocument | **TreeNode** | Entity: DocumentContent, ContentRevision, ShareLink；VO: NodeType, Title, SortOrder, ContentVersion, BodyMd | 原 Tree+Document 合并叙述；表仍 1:1 拆分 |
| Audit | **AuditLog** | — | 追加写；亦可作基础设施 |

`access`（AccessPolicy）为**领域服务**：只读 Member / Share / Node / Kb，无独立表。

### 2.2 值对象（轻量，不建表）

| VO | 规则 |
|----|------|
| MobileE164 | `^\+[1-9]\d{6,14}$`，存 `users.mobile_e164` |
| NodeType | `folder` \| `doc` |
| MemberRole | `owner` \| `editor` \| `reader` |
| ContentVersion | 整数 ≥ 1 |
| BodyMd | UTF-8；`byte_length` ≤ 2097152 |

### 2.3 对象关系（概念）

```
User 1 ── 1 Workspace
Workspace 1 ── * KnowledgeBase
KnowledgeBase 1 ── * KbMember * ── 1 User     # 权限真源；每库恰好一个 owner
KnowledgeBase 1 ── * TreeNode
TreeNode (parent) 1 ── * TreeNode (children)   # parent 可为 folder 或 doc
TreeNode (type=doc) 1 ── 0..1 DocumentContent
TreeNode (type=doc) 1 ── * ContentRevision      # 快照挂 node_id
TreeNode (type=doc) 1 ── 0..1 ShareLink(active) # enabled=true 至多一条
User 1 ── * AuditLog (actor)
```

### 2.4 聚合边界与事务

| 聚合根 | 事务边界示例 |
|--------|----------------|
| User + Workspace | 注册：创建用户与默认空间同一事务 |
| KnowledgeBase | 创建库 + **强制** insert owner 成员；**删库级联**（§6.1） |
| TreeNode | 创建（doc 同事务插 Content）；move；软删；save/overwrite/save-as；分享开关 |
| AuditLog | 与业务写同一事务提交（P0 事件） |

另存副本 = 新 TreeNode + Content（同事务），**不**改源 Content。

## 3. 标识与通用约定

### 3.1 主键

| 项 | 约定 |
|----|------|
| 类型 | `CHAR(26)` ULID（字典序近似时间有序） |
| 生成 | API 进程内生成，不依赖 DB sequence |
| 对外 | 知识库 `kbId`、节点 `nodeId` 等均为 ULID 字符串 |

### 3.2 时间戳

- `created_at` / `updated_at`：`TIMESTAMPTZ`，默认 `now()`  
- 应用层也可写，以 DB 默认兜底  
- API 对外 ISO-8601 UTC  

### 3.3 软删除

| 表 | 策略 |
|----|------|
| `tree_nodes` | `deleted_at` 可空；查询默认 `IS NULL` |
| `knowledge_bases` | 同上 |
| `users` | MVP 可不做注销；预留 `deleted_at` 可选 |
| `document_contents` | **不单独软删**；随 node 软删后不可访问 |
| `share_links` | 仅用 **`enabled=false` 关闭**（不使用 `revoked_at`）；node 或 kb 软删后业务拒绝 |
| `audit_logs` / `content_revisions` | **不删**（追加型） |
| 物理删除 | MVP **禁止**业务路径物理删主数据；FK 默认 `ON DELETE RESTRICT` |

### 3.4 枚举

PostgreSQL 使用 `TEXT` + `CHECK`，或 Prisma `enum` 映射。本文用 CHECK 语义描述。

### 3.5 正文编码与大小

- 编码：**UTF-8**  
- 上限：`byte_length(body_md) ≤ 2097152`（2MB），API 返回 413

## 5. 表结构明细

> 下列 `NOT NULL` / 默认值 / 索引为 **MVP 必须**；标注 P1 的列可延后迁移。

### 5.1 `users`

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| id | CHAR(26) | PK | ULID |
| mobile_e164 | VARCHAR(20) | NOT NULL, **UNIQUE** | E.164，如 `+8613800138000` |
| mobile_verified | BOOLEAN | NOT NULL DEFAULT true | 注册成功即为 true |
| nickname | VARCHAR(64) | NOT NULL | 可默认「用户xxxx」 |
| email | VARCHAR(255) | NULL | P1 资料；不做唯一强制 |
| password_hash | VARCHAR(255) | NULL | P1；argon2/bcrypt |
| avatar_url | VARCHAR(1024) | NULL | P1 |
| created_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |
| updated_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |
| deleted_at | TIMESTAMPTZ | NULL | 预留注销 |

**索引**：

- `UNIQUE (mobile_e164)`  
- 可选：`email` 在非空时唯一 → `UNIQUE` 部分索引 `WHERE email IS NOT NULL`（P1）

**业务规则**：

- 未验证手机号不得存在可登录用户（MVP 注册路径保证 verified=true）。

| owner_user_id | CHAR(26) | NOT NULL, **UNIQUE**, FK → users(id) | 个人空间 1:1 |
| name | VARCHAR(128) | NOT NULL | 默认「我的空间」 |
| created_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |

**索引**：`UNIQUE (owner_user_id)`

### 5.3 `knowledge_bases`

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| id | CHAR(26) | PK | 对外 kbId |
| workspace_id | CHAR(26) | NOT NULL, FK → workspaces(id) | |
| owner_user_id | CHAR(26) | NOT NULL, FK → users(id) | 冗余便于鉴权，与创建者一致 |
| name | VARCHAR(128) | NOT NULL | trim 后非空 |
| description | VARCHAR(2000) | NULL | |
| visibility | TEXT | NOT NULL DEFAULT 'private' | **MVP CHECK 仅 `'private'`**；勿写 workspace |
| created_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |
| updated_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |
| deleted_at | TIMESTAMPTZ | NULL | 软删（级联见 §6.1） |

**索引**：

- `(workspace_id) WHERE deleted_at IS NULL`  
- `(owner_user_id) WHERE deleted_at IS NULL`  
- `(updated_at DESC)` 可选，用于最近列表  

**规则（冻结 · 鉴权双写）**：

- `owner_user_id` = **冗余投影**（列表展示、排错），**不是** access 回退源。  
- 创建 KB **同一事务必须**：

```sql
INSERT INTO knowledge_bases (..., owner_user_id) VALUES (..., $uid);
INSERT INTO kb_members (id, knowledge_base_id, user_id, role)
VALUES (..., $kbId, $uid, 'owner');
```

- 转让 owner（P1）：单事务更新 `kb_members` + `owner_user_id`，保持一致。  
- **禁止**只写 `owner_user_id` 不写 members。

### 5.4 `kb_members`（权限唯一真源；P1 多成员扩展同表）

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| id | CHAR(26) | PK | |
| knowledge_base_id | CHAR(26) | NOT NULL, FK → knowledge_bases(id) | |
| user_id | CHAR(26) | NOT NULL, FK → users(id) | |
| role | TEXT | NOT NULL | CHECK IN ('owner','editor','reader') |
| created_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |
| updated_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |

**约束**：

- `UNIQUE (knowledge_base_id, user_id)`  
- **每库恰好/至多一个 owner**（迁移必建）：

```sql
CREATE UNIQUE INDEX uq_kb_one_owner
ON kb_members (knowledge_base_id)
WHERE role = 'owner';
```

**索引**：

- `(user_id, knowledge_base_id)` — 我的库列表  
- `(knowledge_base_id, role)`  

**MVP 策略（冻结）**：

- 迁移即建表；**access 只查本表**，无成员行 → 视为无权限（404）。  
- 不实现「回退 owner_user_id」。

### 5.5 `tree_nodes`

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| id | CHAR(26) | PK | 对外 **nodeId** |
| knowledge_base_id | CHAR(26) | NOT NULL, FK → knowledge_bases(id) | |
| parent_id | CHAR(26) | NULL, FK → tree_nodes(id) | NULL=库根；**可为 folder 或 doc** |
| type | TEXT | NOT NULL | CHECK IN ('folder','doc') |
| title | VARCHAR(512) | NOT NULL | 允许库内重名 |
| sort_order | INT | NOT NULL DEFAULT 0 | 同父下排序，升序；分配见下 |
| created_by | CHAR(26) | NOT NULL, FK → users(id) | |
| updated_by | CHAR(26) | NULL, FK → users(id) | |
| created_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |
| updated_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |
| deleted_at | TIMESTAMPTZ | NULL | |

**索引**：

| 名 | 定义 | 用途 |
|----|------|------|
| idx_nodes_kb_parent_sort | `(knowledge_base_id, parent_id, sort_order) WHERE deleted_at IS NULL` | 拉子列表/建树 |
| idx_nodes_parent | `(parent_id) WHERE deleted_at IS NULL` | 子节点存在性（删除校验） |

> **标题搜索**：MVP 使用 `ILIKE '%q%'`，**前导通配无法有效利用 btree(title)**；单库 ≤5k 可接受 seq scan。`pg_trgm` GIN **一期再上**，不阻塞 MVP。不必为「假装加速模糊搜」强建 title 索引。

**sort_order 分配（冻结）**：

```sql
-- 同父下新建：间隔 1000，减少后续重排
COALESCE(MAX(sort_order), 0) + 1000
WHERE knowledge_base_id = $kb AND parent_id IS NOT DISTINCT FROM $parent
  AND deleted_at IS NULL
```

**完整性（应用层强制 + 单测必覆盖）**：

1. `parent_id` 若非空：parent 同 `knowledge_base_id`、`deleted_at IS NULL`。  
2. **禁止环**：move 时沿新 parent 上溯。  
3. 删除：不存在 active 子节点。  
4. `type='folder'` 不得有 `document_contents`。  
5. `type='doc'` 创建时 **同事务** 插入 `document_contents`。  
6. **不可变字段（冻结）**：禁止 `UPDATE type`；禁止 `UPDATE knowledge_base_id`。  
7. **move 仅允许**改 `parent_id` + `sort_order`，且 `newParent.knowledge_base_id = node.knowledge_base_id`（根则仍为本库）。

**可选 DB 增强（非 MVP 必须）**：触发器校验 parent 同库；环检测保持应用校验。

### 5.6 `document_contents`

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| node_id | CHAR(26) | **PK**, FK → tree_nodes(id) ON DELETE RESTRICT | 1:1 doc 节点 |
| body_md | TEXT | NOT NULL DEFAULT '' | Markdown；应用层 ≤ 2MB |
| content_version | INT | NOT NULL DEFAULT 1 | 单调递增，从 1 起 |
| updated_by | CHAR(26) | NULL, FK → users(id) | |
| created_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |
| updated_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |

**约束**：

- `CHECK (content_version >= 1)`  
- 应用层：仅当 `tree_nodes.type='doc'` 可插入  

**乐观锁 — 正常保存** `PUT .../content` body: `{ expectedVersion, bodyMd }`：

```sql
UPDATE document_contents
SET body_md = $1,
    content_version = content_version + 1,
    updated_by = $2,
    updated_at = now()
WHERE node_id = $3
  AND content_version = $4;  -- expectedVersion
-- affected = 1 成功；0 → 再读：无行/node 删 404，否则 409
```

**乐观锁 — 强制覆盖** `POST .../content/overwrite` body: `{ baseVersion, bodyMd }`（**冻结**）：

- `baseVersion` = 冲突面板展示的 **serverVersion**（用户决策所基于的版本）。  
- **禁止**无 version 谓词的裸 UPDATE（防并发双覆盖丢字）。

```sql
-- 见 §9.5 完整事务
UPDATE document_contents
SET body_md = $body, content_version = content_version + 1, ...
WHERE node_id = $id AND content_version = $baseVersion;
-- affected = 0 → 409 DOC_VERSION_CONFLICT（需重新加载后再覆盖）
```

**大小限制**：API 校验 UTF-8 字节数 ≤ 2097152，不单靠 DB。

### 5.8 `share_links`

| 列 | 类型 | 约束 | 说明 |
|----|------|------|------|
| id | CHAR(26) | PK | |
| token | VARCHAR(64) | NOT NULL, **UNIQUE** | 高熵；MVP 存明文 |
| node_id | CHAR(26) | NOT NULL, FK → tree_nodes(id) | 必须 type=doc |
| enabled | BOOLEAN | NOT NULL DEFAULT true | **唯一状态开关** |
| expires_at | TIMESTAMPTZ | NULL | P1；NULL=直至关闭 |
| created_by | CHAR(26) | NOT NULL, FK → users(id) | |
| created_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |
| updated_at | TIMESTAMPTZ | NOT NULL DEFAULT now() | |

> **已移除 `revoked_at`**（避免与 `enabled` 双状态不一致）。关闭 = `enabled=false`。

**索引**：

- `UNIQUE (token)`  
- **每文档至多一条启用中的分享**（迁移必建）：

```sql
CREATE UNIQUE INDEX uq_share_one_active
ON share_links (node_id)
WHERE enabled = true;
```

**Token 生成**：CSPRNG ≥ 128 bit，URL-safe base64/hex，长度 ≤ 64。  

**MVP 存储**：明文 token（依赖 DB 与备份权限）；一期可改 hash。

**有效性校验（冻结 SQL 语义）**：

```sql
SELECT s.*, n.title, c.body_md, c.content_version
FROM share_links s
JOIN tree_nodes n ON n.id = s.node_id
JOIN knowledge_bases k ON k.id = n.knowledge_base_id
JOIN document_contents c ON c.node_id = n.id
WHERE s.token = $token
  AND s.enabled = true
  AND (s.expires_at IS NULL OR s.expires_at > now())
  AND n.deleted_at IS NULL
  AND k.deleted_at IS NULL
  AND n.type = 'doc';
```

关闭分享：`enabled=false` + audit `share.disable`。  
开启：若已有 enabled 行则先禁用或更新同一行（满足部分唯一）。

## 6. 状态机

### 6.1 知识库删除（级联 · 冻结）

```
active (deleted_at IS NULL)
    │ DELETE /kbs/:kbId （单事务级联）
    ▼
deleted + 全部 active 子节点软删 + 相关 share 禁用
```

**事务步骤（必须）**：

```text
1. access.assert(user, kb, admin)   -- 仅 role=owner
2. UPDATE knowledge_bases SET deleted_at=now()
   WHERE id=$kbId AND deleted_at IS NULL
3. UPDATE tree_nodes SET deleted_at=now()
   WHERE knowledge_base_id=$kbId AND deleted_at IS NULL
4. UPDATE share_links SET enabled=false, updated_at=now()
   WHERE node_id IN (
     SELECT id FROM tree_nodes WHERE knowledge_base_id=$kbId
   ) AND enabled=true
5. INSERT audit_logs action=kb.delete
```

- **不**要求库先空；**不**物理删 content/revisions（运维可恢复）。  
- MVP 无恢复 API。  
- 业务读路径必须 `k.deleted_at IS NULL`（含分享，见 §5.8）。

### 6.2 树节点

```
active ──delete──► deleted
         ▲ 要求无 active 子节点
```

### 6.3 文档正文版本

```
version = n
  │ PUT expectedVersion=n 成功
  ▼
version = n+1

version = n
  │ POST overwrite baseVersion=n 成功
  ▼
revision(body@n) → version = n+1

version = n
  │ overwrite baseVersion 过期（已是 n+1）
  ▼
409 CONFLICT
```

无「草稿/已发布」双状态（P1）；MVP 仅 `body_md` 当前真源。

### 6.4 分享

```
enabled=true  ──disable──► enabled=false
     │
     └── expires_at 过期（P1）→ 业务视为无效
```

## 7. 访问控制的数据读路径（权限真源 = kb_members）

`access` 不落库；**禁止**用 `owner_user_id` 回退鉴权。

### 7.1 读文档 `nodeId`

1. 若 actor = share token：按 §5.8 完整 SQL 命中 → **仅 read 该 node 正文**  
2. 若 actor = user：  
   - 加载 node + kb（**二者** `deleted_at IS NULL`）  
   - 查 `kb_members` where kb_id + user_id  
   - role ∈ owner/editor/reader → allow read  
   - **无行 → deny 404**（即使 `owner_user_id` 碰巧等于该用户——数据应已双写，缺行视为数据错误/拒绝）

### 7.2 写文档 / 树

- `kb_members.role` ∈ `owner` | `editor`  
- reader / share → deny  

### 7.3 admin（成员管理、删库）

- `kb_members.role = owner` 仅  

### 7.4 我的知识库列表

```sql
SELECT kb.*
FROM knowledge_bases kb
JOIN kb_members m ON m.knowledge_base_id = kb.id
WHERE m.user_id = $uid AND kb.deleted_at IS NULL;
```

## 9. 关键查询与写入 SQL 形态

### 9.1 拉取整树

```sql
SELECT id, parent_id, type, title, sort_order, updated_at
FROM tree_nodes
WHERE knowledge_base_id = $kbId
  AND deleted_at IS NULL
ORDER BY parent_id NULLS FIRST, sort_order, id;
```

应用层 O(n) 建树。

### 9.2 标题搜索

```sql
SELECT id, parent_id, type, title
FROM tree_nodes
WHERE knowledge_base_id = $kbId
  AND deleted_at IS NULL
  AND title ILIKE '%' || $q || '%'   -- 应用层转义 % _
ORDER BY updated_at DESC
LIMIT 50;
```

### 9.3 是否存在未删子节点

```sql
SELECT 1 FROM tree_nodes
WHERE parent_id = $nodeId AND deleted_at IS NULL
LIMIT 1;
```

### 9.4 创建 doc 节点（事务）

```sql
BEGIN;
INSERT INTO tree_nodes (... type='doc' ...);
INSERT INTO document_contents (node_id, body_md, content_version) VALUES ($id, '', 1);
COMMIT;
```

### 9.5 Overwrite（事务 · 含 baseVersion）

```sql
BEGIN;
-- 已 assert access.write；body 长度已校验

INSERT INTO content_revisions (
  id, node_id, version, body_md, reason, created_by, created_at
)
SELECT $revId, c.node_id, c.content_version, c.body_md,
       'overwrite_on_conflict', $uid, now()
FROM document_contents c
JOIN tree_nodes n ON n.id = c.node_id
WHERE c.node_id = $id
  AND c.content_version = $baseVersion
  AND n.deleted_at IS NULL
  AND n.type = 'doc';
-- INSERT 0 行 → 说明版本已变或不存在 → ROLLBACK → 409 或 404

UPDATE document_contents
SET body_md = $body,
    content_version = content_version + 1,
    updated_by = $uid,
    updated_at = now()
WHERE node_id = $id
  AND content_version = $baseVersion;
-- affected must be 1

INSERT INTO audit_logs (
  id, actor_user_id, action, resource_type, resource_id, metadata, request_id, created_at
) VALUES (
  $auditId, $uid, 'doc.overwrite_on_conflict', 'node', $id,
  jsonb_build_object('baseVersion', $baseVersion, 'newVersion', $baseVersion + 1),
  $requestId, now()
);
COMMIT;
```

### 10.1 必须手写 SQL 的约束清单（Prisma 难表达 · 首迁 DoD）

首迁 `migrate deploy` 后须存在：

| 对象 | SQL 要点 |
|------|----------|
| `uq_kb_one_owner` | `UNIQUE (knowledge_base_id) WHERE role = 'owner'` on `kb_members` |
| `uq_share_one_active` | `UNIQUE (node_id) WHERE enabled = true` on `share_links` |
| CHECK type | `tree_nodes.type IN ('folder','doc')` |
| CHECK role | `kb_members.role IN ('owner','editor','reader')` |
| CHECK visibility | `knowledge_bases.visibility = 'private'`（MVP） |
| CHECK reason | `content_revisions.reason = 'overwrite_on_conflict'` |
| CHECK version | `document_contents.content_version >= 1` |
| 部分索引树 | `(knowledge_base_id, parent_id, sort_order) WHERE deleted_at IS NULL` |
| 部分索引库 | `(workspace_id) WHERE deleted_at IS NULL` 等 §5 所列 |

`schema.prisma` 中的 `@@index` 可逐步对齐；**部分唯一以 SQL 为准**。
