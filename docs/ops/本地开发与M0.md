# 本地开发与 M0 基线

## 前置

- Node.js **22**（`.nvmrc`）
- pnpm 9（`packageManager`）
- Docker（Compose：PostgreSQL 16 + Redis 7）

## 一键依赖

```bash
cp .env.example .env
# 确认仓库根目录 .env 中 DATABASE_URL 与 Compose 一致，默认：
# postgresql://yuque:localdevonly@localhost:5432/yuque1?schema=public

pnpm install
pnpm compose:up   # PG user/pass: yuque / localdevonly（仅本地）

# 推荐（自动加载根目录 .env，避免 P1012）：
pnpm db:migrate
pnpm db:generate
```

### DATABASE_URL 常见错误

| 报错 | 原因 | 处理 |
|------|------|------|
| `P1012 Environment variable not found: DATABASE_URL` | 根目录无 `.env`，或用了裸 `exec prisma` 未加载 env | `cp .env.example .env` 后用 `pnpm db:migrate` |
| `P1001 Can't reach database server` | Compose 未起或当前环境访问不到 `localhost:5432`（DinD/嵌套网络常见） | 1) `pnpm compose:up` 确认 healthy<br>2) 宿主探测：`node -e "require('net').connect(5432,'127.0.0.1',()=>console.log('ok')).on('error',e=>console.error(e))"`<br>3) 若 ECONNREFUSED：用 **`pnpm db:migrate:docker`**（在 compose 网络内对 `postgres` 主机迁移） |

> 不要使用：`pnpm --filter @yuque1/api exec prisma migrate deploy`（不会读 monorepo 根 `.env`）。

### Nest 与 Prisma 的 env 一致性

- Prisma CLI：`apps/api/scripts/run-prisma.cjs` 加载根 `.env`
- Nest 运行时：`apps/api/src/load-env.ts` 在 `main.ts` 入口同样加载
- 请始终在**仓库根**维护 `.env`

## 启动

```bash
# 终端 1
pnpm dev:api

# 终端 2
pnpm dev:web
```

- Web: http://localhost:5173 （代理 `/api` → API）
- API: http://localhost:3000/api/v1/health

## 质量命令

```bash
pnpm run ci
```

含：Node 检查、docs 检查、prisma generate、typecheck、unit test、build。
